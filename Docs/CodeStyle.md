# Code Style

## Загальні принципи

- використовувати серіалізовані приватні поля для налаштувань у `Inspector`
- перевіряти критичні залежності в `Awake()`
- вимикати компонент, якщо валідація конфігурації не пройдена
- віддавати перевагу коротким компонентам з однією відповідальністю

## Поточні патерни в коді

- `RequireComponent` для явних залежностей
- `Guard.Expect(...)` для runtime validation
- групування полів через `[Header(...)]`
- зберігання налаштувань вводу через `InputActionReference`

## Рекомендації для продовження

- не змішувати actor input, предметну логіку і stack assembly в одному класі
- нові gameplay systems додавати окремими компонентами
- зберігати однорідний стиль назв для prefab, material і script assets
- уникати прихованих runtime dependency, якщо їх можна виразити через `RequireComponent`

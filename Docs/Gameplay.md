# Gameplay

## Огляд

Поточна версія проєкту є sandbox-прототипом, у якому гравець керує персонажем від першої особи, взаємодіє з фізичними об'єктами, збирає бургер зі стекованих інгредієнтів і користується базовим HUD для прицілювання в центр екрана.

## Керування

- `WASD` або `Left Stick` - рух
- `Mouse Delta` або `Right Stick` - поворот камери/персонажа
- `Space` або `Gamepad South` - стрибок
- `Left Mouse Button` або `Right Shoulder` - взяти / відпустити предмет
- `Mouse Scroll` або `Triggers` - змінити дистанцію утримання предмета
- `Right Mouse Button` або `Left Shoulder` - кинути предмет

## Основні системи

### Рух персонажа

- `IaMovement` задає горизонтальний рух персонажа через `Rigidbody.linearVelocity`
- `IaRotation` відповідає за yaw тіла та pitch камери
- `IaJump` виконує стрибок лише при наявності контакту з землею

### Взаємодія з предметами

- `IaHoldObject` шукає `HoldableObject` через raycast із центру екрана
- при підборі предмет лишається динамічним `Rigidbody` і пружно тягнеться до `holdPoint`
- колізії між гравцем і утримуваним об'єктом тимчасово ігноруються, але колізії зі світом лишаються активними
- `IaZoomObject` змінює локальну позицію `holdPoint` по осі `Z`, а предмет доганяє нову дистанцію через physics-follow
- `IaThrowObject` спочатку завершує hold-режим, а потім додає імпульс уперед від камери

### HUD

- `UI_GameplayHUD` відображається як screen-space overlay canvas
- у центрі HUD розміщений `Image` із sprite `aim.png`, який слугує crosshair для взаємодії з предметами

### Збірка бургера

- `StackableObject` має `topPoint` і `bottomPoint` для точного стекування
- `StackHolder` приймає новий інгредієнт у trigger-зону
- після додавання `stackPoint` переноситься на верхню точку останнього елемента
- `BurgerData` зберігає поточний список інгредієнтів у стеку

## Поточний gameplay loop

1. Знайти тарілку або інгредієнт у сцені.
2. Підібрати об'єкт.
3. Піднести його до точки зборки.
4. Дочекатися входу в trigger stack holder.
5. Отримати зібраний стек із кількох інгредієнтів.

## Обмеження поточної версії

- немає перевірки валідності рецепта
- немає системи замовлень, таймерів або розширеного UI поза базовим crosshair HUD
- немає інтеграції Mirror / Steamworks у runtime
- немає синхронізації stack state між клієнтами
